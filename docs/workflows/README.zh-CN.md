# Codex 工作流

## 工作流 1：读代码，不修改

```text
请阅读项目结构，找出入口文件、测试命令和关键配置。不要改文件。
```

用在：新项目接手、代码审查前、先让 Codex 建立上下文。

## 工作流 2：补测试再实现

```text
请先为目标行为写一个失败测试，运行确认失败，再实现最小修改，最后运行相关测试。
```

用在：bugfix、新功能、小型重构。

## 工作流 3：安全修改配置

```text
请检查配置文件并提出修改计划。不要打印任何密钥，不要修改项目外文件。
```

用在：CI、构建、工具链、环境配置。

## 工作流 4：审查 diff

```text
请审查当前 diff，优先指出 bug、风险、行为回归和缺失测试。
```

用在：提交前自查、PR 前检查。
