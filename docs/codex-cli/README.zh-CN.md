# Codex CLI 使用

## 目标

让 Codex 在本地终端里读代码、改代码、跑测试。先从低风险任务开始。

## 官方来源

- [Codex CLI docs](https://developers.openai.com/codex/cli)
- [openai/codex](https://github.com/openai/codex)

## 基本顺序

1. 进入一个 Git 仓库。
2. 确认工作区干净，或至少知道哪些改动是自己的。
3. 启动 Codex，让它先读关键文件。
4. 要求它说明计划和会运行的命令。
5. 只授权你理解的修改和命令。
6. 要求它运行项目已有测试。
7. 审查 diff，再决定是否提交。

## 示例任务

```text
请阅读这个仓库，找出测试命令和主要入口文件。不要改文件。先给我一个简短计划。
```

```text
请为 utils/date.ts 增加单元测试。先写测试，运行确认失败，再实现最小修改，最后运行相关测试。
```

## 写教程时要有

每篇 Codex CLI 教程都要写清楚：

- 适用平台和版本。
- 配置文件路径。
- 预期命令输出。
- 常见失败模式。
- 权限和安全提示。
- 最后验证日期。
