## 使用顺序

不要一开始就装很多扩展。推荐顺序：

1. 先跑通 Codex CLI。
2. 写好 `AGENTS.md`。
3. 确认权限和审批边界。
4. 再按任务引入 MCP、Skills、Plugins、Subagents。
5. 需要中文系统路线时，再参考 Codex 橙皮书和 CodexGuide，并把功能、价格、认证、安全策略回到官方资料交叉确认。

## 收录提醒

生态资源不等于背书。任何需要 token、浏览器、数据库、云账号、本地文件写入的工具，都必须先看源码、权限范围和数据流。

## 中文高热资源

- [bozhouDev/codex-orange-book](https://github.com/bozhouDev/codex-orange-book) - Codex 橙皮书，适合中文用户按 PDF/长文方式系统阅读。
- [freestylefly/CodexGuide](https://github.com/freestylefly/CodexGuide) - CodexGuide 开源仓库，适合查看中文教程站背后的内容结构。
- [CodexGuide.ai](https://codexguide.ai/) - 教程网站，覆盖 Codex 桌面 App、CLI、IDE、Cloud、AGENTS.md、沙盒审批、排障和实战案例。

这些资源热度高、中文友好，但仍属于社区资源。涉及可用性、订阅、认证、权限和安全策略时，以 OpenAI 官方资料为准。
